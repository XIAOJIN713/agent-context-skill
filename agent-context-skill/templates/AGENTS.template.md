## 4. Agent 工作规则（Working Rules）

- 重要工作开始前，先阅读本文件。
- 依赖具体事实前，直接阅读相关项目源资料。
- 优先做小而可审查的改动。
- 新增说明应放在最接近其影响范围的位置。
- 任务状态发生实质变化或工作暂停时，更新相关交接文档（handoff）。
- 如果 handoff 或项目源资料留下重大不确定性，先确认再做大范围修改。

## 5. 常见误区 / 禁止事项（Do Not Do）

- 不要把项目源资料正文复制到本文件。
- 不要在这里记录单个会话（session）的临时进度。
- 不要添加不存在的项目源资料类型或路径。
- 不要把本文件变成完整仓库（repository）索引、资料库或任务日志。
- 项目结构变化后，不要保留过期说明。
- 不要重复已经存在于项目文件、说明文件、配置或 handoff 中的细节。

## 6. 交接规则（Handoff Rules）

- 将 handoff 存放在 `agent-context/handoff/`。
- 每个活跃任务、线程或会话主题使用一个 handoff。
- 每个 handoff 只保留继续工作所需的最小状态。
- Handoff 应包含当前状态、已确定结论、卡点或待确认问题、下一步和关键文件。
- 不要把本文件中的全局规则复制到 handoff。
- 不要把项目源资料正文复制到 handoff。
- 当 handoff 已完成、重复、过期或会误导后续工作时，及时修订或移除。
