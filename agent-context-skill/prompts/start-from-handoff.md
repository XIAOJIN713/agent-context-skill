# 提示词（Prompt）：从 Handoff 接续

新会话（session）从指定交接文档（handoff）接续任务时，使用这段提示词。

```text
你正在从已有项目 handoff 接续一个任务。

输入：
- 项目上下文文件：`agent-context/AGENTS.md`
- Handoff 文件：`{指定 handoff 的路径}`

执行要求：
1. 先读取 `agent-context/AGENTS.md`。
2. 再读取指定 handoff 文件。
3. 再按 handoff 中列出的关键文件，读取下一步所必需的项目源资料（source material）。
4. 如果 handoff 中的文件缺失或疑似过期，明确说明，并先搜索项目，不要直接假设替代路径。
5. 如果 handoff 与当前项目源资料冲突，先报告冲突，并以当前项目源资料为准，除非用户另有说明。
6. 用简短条目复述当前状态。
7. 用 3-6 条说明下一步计划。
8. 说明接下来需要查看或编辑哪些文件。
9. 在用户确认接续计划前，不要直接进行大规模修改。
10. 用户确认后，继续做聚焦工作；任务状态发生实质变化时，更新 handoff。

不要：
- 把 `AGENTS.md` 复制进 handoff。
- 把项目源资料正文复制进对话，除非确实需要短摘录来说明问题。
- 将过期 handoff 视为比当前项目源资料更权威。
```
